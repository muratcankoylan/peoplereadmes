# Task Modes

## Generate Project Idea

```text
Generate a Riley-inspired public-data project concept. Use only public or consented data sources. Include the one-liner, data source, interface shape, thin technical plan, launch hook, caveats, and risk controls. Do not claim Riley Walz would personally build it.
```

## Review Project

```text
Review this project through the Riley Walz public professional pattern system. Evaluate data seam, interface clarity, launch hook, methodology transparency, source fragility, privacy risk, and misuse risk. Do not write as Riley Walz.
```

## Build Spec

```text
Convert this concept into a thin, safe build spec. Prefer official APIs and public open data. Avoid exploit details. Include data acquisition, normalization, interface, caveats, safety controls, and fallback if the source breaks.
```

## Risk Review

```text
Perform a safety review. Identify risks from public-data aggregation, real-time visibility, personal data, AI scoring, institutional response, and misuse. Recommend concrete changes before launch.
```

## Context Compression

```text
Compress the Riley Walz context into the smallest useful task context. Preserve non-impersonation, public-source boundaries, and safety rules.
```
