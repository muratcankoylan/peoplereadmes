# Tacit Knowledge

This file captures inferred public professional heuristics from Riley Walz's projects. It does not claim access to private thinking.

## Latent Interface Instinct

The strongest pattern is seeing latent interfaces inside existing systems. A city portal, public API, platform search quirk, product price endpoint, reservation system, camera feed, or old filename convention already contains a story. The work is to make it visible.

```text
system behavior -> exposed residue -> simple inference -> public interface -> reaction
```

## Thinking Heuristics

### Use Data As Medium

The dataset is the project. The interface is a frame around the dataset.

### Make One Thing Obvious

Avoid broad dashboards. Pick one question:

- What music is playing here?
- Where did tickets just get written?
- What forgotten videos match this filename?
- How do prices differ by location?

### Prefer Thin Proof Over Durable Platform

The public artifact can be small if the idea is sharp. Durability is optional when the project is an intervention.

### Launch To Learn

The release is not the end of the experiment. Traffic, press, backlash, takedown, remixing, and confusion are feedback.

### Explain The Trick

A methodology page or caveat increases credibility and lets technically curious users understand the system.

### Treat Publicness Carefully

Public data can still create harm when aggregated, ranked, delayed too little, or wrapped in a new interface.

## Practical Checklist

Before building a concept:

1. What public data residue exists?
2. What surprising inference can be made?
3. Can the payoff fit on one screen?
4. What familiar interface makes it legible?
5. What can go wrong if it goes viral?
6. What safety control preserves the insight while reducing harm?
7. What does the page show if the source breaks?
